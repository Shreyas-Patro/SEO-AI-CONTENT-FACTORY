Documentation

How to use the pipeline.

A step-by-step walkthrough of producing your first cluster. Estimated time from "I have a topic" to "articles are live": about 90 minutes including review.

{# ─── STEP 1 ───────────────────────────────────────── #}
Step 1

Sign in and start a run.

Go to /login, sign in, then enter a topic in the sidebar — for example, "Sarjapur Road". Click Start New Run.

Tip
Topics should be specific enough to support 8–15 articles. "Real estate" is too broad; "Sarjapur Road" is right.
{# ─── STEP 2 ───────────────────────────────────────── #}
Step 2

Run Layer 1 — discovery.

Click ▶ Layer 1. Three agents run in sequence:

  • Trend Scout — pulls current SERP, related searches, PAA boxes.
  • Competitor Spy — audits MagicBricks, NoBroker, Housing.com top 10.
  • Keyword Mapper — groups keywords by intent and difficulty.

About 2–3 minutes. You'll see live console output stream in. Each agent's full input/output is inspectable from the agent grid.

{# ─── STEP 3 ───────────────────────────────────────── #}
Step 3

Approve the gate.

Layer 1 ends at the approval gate. Open the keyword map, sanity check the intent groupings, then click ✅ Approve. If something's off, click ✗ Reject — the run is cancelled. Edit any agent's output JSON in place if you want to nudge it.

{# ─── STEP 4 ───────────────────────────────────────── #}
Step 4

Run Layer 2 — architecture.

Click ▶ Layer 2. Three more agents run:

  • Content Architect — designs the cluster (1 hub, N spokes, M sub-spokes).
  • FAQ Architect — generates the AEO-optimised question set.
  • Research Prompt Generator — writes the brief for each spoke.
{# ─── STEP 5 ───────────────────────────────────────── #}
Step 5

Ingest research.

Open /ingestion. Drop your research documents (.md, .pdf, .docx, .txt) — they're chunked, embedded into ChromaDB, and added to the knowledge graph. The Lead Writer queries this corpus for every claim.

Important
Without ingested research, the Fact Verifier will reject most claims and the quality loop will hit its iteration cap. Ingest before running L3.
{# ─── STEP 6 ───────────────────────────────────────── #}
Step 6

Run Layer 3 — writing & quality.

Click ▶ Layer 3. For each article in the cluster, the pipeline runs:

  1. Lead Writer drafts.
  2. Fact Verifier checks every claim against ingested research.
  3. Brand Auditor scores tone (target ≥ 6.5/10).
  4. Rewriter fixes whatever didn't pass — up to two iterations.
  5. Meta Tagger adds slug, schema, OG tags, canonical URL.

Articles that fail the quality loop after 2 iterations are flagged needs_human_review. Open them in the Articles tab, edit in-browser, save.

{# ─── STEP 7 ───────────────────────────────────────── #}
Step 7

Interlink.

Open the Interlink tab. Click Run Cluster Pass. The Link Engine extracts canonical anchor phrases, finds candidates across all articles in this cluster, and proposes links with confidence scores.

Approve or reject each candidate. Click Inject Approved → when you're done. Markdown gets rewritten in place with backups in .md.bak.

Run the Global Pass if you want to interlink against every previously-published article, not just this cluster.

{# ─── STEP 8 ───────────────────────────────────────── #}
Step 8

Export.

From the Articles tab:

  • Download All (publish-ready) — Hugo/Astro frontmatter + JSON-LD schema.
  • Download for Link Engine — minimal frontmatter, ready to round-trip.
  • Per-article .md — single article, both formats available.
